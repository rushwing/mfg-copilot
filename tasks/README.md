# Tasks

This directory is the repo-native source of truth for implementation-facing requirement content.

Structure:
- `phases/`: phase contracts and scope boundaries
- `features/`: feature stories in `REQ-xxx.md` format
- `archive/`: completed or cancelled work items

Working model:
- edit requirement content here
- review and discussion happen through GitHub PRs
- architecture rationale stays in `docs/architecture/`
- requirement management rules live in `harness/requirement-standard.md`
