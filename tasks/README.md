# Tasks Home

This directory is the repo-native source of truth for implementation-facing requirements.

Use the indexes below instead of opening individual `REQ` files at random.

## Read In This Order

1. [Phase Index](./phases/README.md) for roadmap boundaries and current sequencing.
2. [Feature Index](./features/README.md) for story-level scope and dependencies.
3. Individual `REQ-xxx` files only when you are implementing or reviewing that story.

## Operating Model

- `tasks/` owns requirement content
- GitHub owns review and merge state
- `docs/architecture/` owns rationale and detailed design
- `harness/requirement-standard.md` defines the story contract

## Directory Map

- `phases/` for phase contracts and scope boundaries
- `features/` for feature stories in `REQ-xxx.md` format
- `archive/` for completed or cancelled work items
