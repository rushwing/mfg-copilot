# User Story

As a project engineer, I want grounded answers to documentation questions so I can move faster without guessing.

# Goal

Deliver the first useful agent workflow in production.

# Deliverables

- doc-copilot runtime path wired through router and orchestrator
- grounded question answering workflow
- citation rendering support in returned payloads
- context-aware handling for project, phase, checkpoint, and station filters

# In Scope

- documentation and checkpoint question answering
- citation rendering
- context-aware prompts for project and phase questions

# Out of Scope

- document execution or approval

# Acceptance Criteria

1. Users can submit documentation and checkpoint questions from the portal and receive responses through the orchestrated agent path.
2. Answers include citations or evidence references derived from normalized retrieval results, and each cited source maps to chunks present in the retrieval response.
3. Responses honor project, phase, checkpoint, or station context when that context is available in the request.
4. The workflow remains read-only and does not invoke execution adapters.
5. Model selection follows configured routing policy rather than agent-local hardcoding.

# Validation Notes

- harness evals should include citation quality and hallucination resistance checks

# Dependency Notes

- Depends on `REQ-004` for retrieval, `REQ-005` for model-routing policy, and `REQ-006` for the portal interaction surface.
