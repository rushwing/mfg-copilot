# User Story

As a manufacturing user, I want a single portal workspace so I can interact with the platform and review past results.

# Goal

Provide the first coherent user experience for platform workflows.

# Deliverables

- request entry workspace in the portal
- task or conversation history view
- result rendering for citations, summaries, and structured artifacts
- baseline empty, loading, and failure states for user-facing workflow views

# In Scope

- chat or request entry surface
- task list or history view
- result rendering for citations, summaries, and artifacts

# Out of Scope

- advanced analytics dashboards

# Acceptance Criteria

1. Users can submit a request from the portal and observe status changes until completion.
2. The portal shows prior tasks or conversation history for the active user context.
3. Result views can render at least citations, summaries, and structured artifacts such as checklists.
4. Empty, loading, and failure states are handled explicitly rather than relying on raw errors.
5. Portal interactions align with shared task and result contracts rather than ad hoc response handling.

# Validation Notes

- Playwright should cover the main browser workflow

# Dependency Notes

- Depends on `REQ-001`, `REQ-002`, and `REQ-003` because the portal experience needs local dev scaffolding, authenticated context, and stable response envelopes.
