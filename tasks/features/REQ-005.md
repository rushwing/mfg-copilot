# User Story

As a platform administrator, I want model selection controlled by policy so cost, safety, and export constraints stay governable.

# Goal

Create the first policy-driven model-routing path for all agent workflows.

# Deliverables

- task class taxonomy
- model profile catalog
- policy override rules for restricted workloads
- documented resolution path from task class to model profile

# In Scope

- task class taxonomy
- model profile mapping
- policy overrides for export-controlled or PII-sensitive work

# Out of Scope

- dynamic optimization based on production telemetry

# Acceptance Criteria

1. Task-to-model routing is resolved through versioned configuration instead of scattered service-local conditionals.
2. Restricted workloads can be forced to approved local model profiles through policy overrides.
3. Services consume resolved profiles or policy decisions rather than hardcoded vendor-specific model names.
4. The configuration shape is stable enough to support additional task classes later without redesign.
5. Policy resolution behavior is testable and documented.

# Validation Notes

- unit tests should cover policy resolution and override behavior

# Dependency Notes

- Depends on `REQ-002` for identity and safety context inputs and on `REQ-003` for shared contract alignment.
