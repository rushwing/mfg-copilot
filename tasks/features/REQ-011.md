# User Story

As an approver, I want a clear approval inbox so I can make safe decisions on execution-ready plans.

# Goal

Turn the existing approval schema into an active workflow.

# Deliverables

- approval inbox portal view
- approval retrieval APIs
- decision submission flow
- audit persistence for approval actions

# In Scope

- approval inbox UI
- approval request retrieval and rendering
- decision submission and audit capture

# Out of Scope

- execution itself

# Acceptance Criteria

1. Pending approvals appear in a dedicated portal view rather than being embedded as incidental task metadata.
2. Approvers can inspect risk, scope, evidence, and summary context before making a decision.
3. Approval decisions are persisted with actor identity, timestamp, and optional reviewer comment.
4. Approval APIs and UI align with the shared approval schema instead of inventing separate payload shapes.
5. Access to approval actions is protected by policy-aware role checks.

# Validation Notes

- Playwright should cover the approval inbox flow
- integration tests should verify approval state transitions, persistence, RBAC behavior, and audit writes

# Dependency Notes

- Depends on `REQ-005` in addition to identity, schema, and portal baselines because approval actions need policy-aware gating and role enforcement.
