# User Story

As a project engineer, I want gap reports and checklists so I know what remains before a checkpoint can pass review.

# Goal

Turn documentation workflows into actionable readiness guidance.

# Deliverables

- gap analysis output shape
- checklist generation output
- readiness summary logic for phase or checkpoint scope
- portal rendering for satisfied, missing, and uncertain items

# In Scope

- prerequisite gap detection
- checklist generation
- readiness summaries by phase or checkpoint

# Out of Scope

- automatic remediation of missing items

# Acceptance Criteria

1. Users can request a readiness or gap analysis for a document, template, or checkpoint scope.
2. Outputs clearly separate satisfied, missing, and uncertain items instead of blending them into one narrative.
3. Checklist artifacts are durable enough to be revisited from task history.
4. Gap outputs are grounded in available template, KB, and contextual data rather than unsupported guesses.
5. Partial or missing context is surfaced explicitly when the system cannot determine readiness confidently.

# Validation Notes

- harness scenarios should include partial and missing-context cases

# Dependency Notes

- Depends on `REQ-008` because gap and checklist logic should evaluate structured deliverables and validation results rather than starting from raw prose.
