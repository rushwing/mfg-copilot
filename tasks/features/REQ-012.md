# User Story

As a deployment engineer, I want structured deployment plans so I can review and execute consistent bring-up workflows.

# Goal

Deliver the planning half of the deploy workflow before broad execution.

# Deliverables

- deploy-agent workflow entry point
- plan generation for supported site and station scopes
- explicit pre-check, risk, and approval requirement outputs
- portal review path for saved deployment plans

# In Scope

- deployment plan generation
- pre-check identification
- risk and prerequisite summaries

# Out of Scope

- broad automatic rollout

# Acceptance Criteria

1. Users can request a deployment plan for explicitly supported station or site contexts.
2. Plans include pre-checks, risks, required approvals, and relevant context assumptions.
3. Deployment plans are stored as reviewable artifacts in the portal rather than transient-only responses.
4. The workflow remains planning-only and does not execute enterprise actions.
5. Model selection and safety constraints follow configured policy.

# Validation Notes

- harness evals should cover plan quality and missing-context handling

# Dependency Notes

- Depends on `REQ-011` because deployment plans must be shaped for later approval review, and on `REQ-005` because planning quality and safety should follow model-routing policy.
