# User Story

As an operations or FA stakeholder, I want structured RCA and escalation outputs so cross-team follow-up is faster and more consistent.

# Goal

Turn triage outputs into durable handoff artifacts.

# Deliverables

- RCA draft generation path
- FA-focused summary rendering
- escalation package output with evidence references
- portal presentation for post-triage artifacts

# In Scope

- RCA draft generation
- FA summary generation
- escalation package rendering

# Out of Scope

- final human sign-off workflows

# Acceptance Criteria

1. Users can generate RCA-oriented and FA-oriented outputs from a completed triage context.
2. Outputs include grounded evidence references, open questions, and explicit uncertainty where needed.
3. Escalation packages are formatted as durable handoff artifacts rather than ad hoc chat text.
4. RCA and FA outputs remain reviewable from the portal after generation.
5. The workflow does not present generated RCA text as final human-approved truth by default.

# Validation Notes

- harness evals should include factuality and evidence-linkage checks

# Dependency Notes

- Depends on `REQ-015` because RCA and escalation outputs should build from grounded triage artifacts rather than bypassing triage.
