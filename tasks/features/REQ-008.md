# User Story

As a project engineer, I want template-conformant drafts so generated outputs are immediately useful instead of raw prose only.

# Goal

Move Doc Copilot from answer generation to structured deliverable generation.

# Deliverables

- template selection or routing logic
- structured draft generation by template section
- schema or required-section validation output
- portal-friendly rendering of generated drafts and validation status

# In Scope

- template selection
- structured section generation
- validation against required sections and template schema

# Out of Scope

- enterprise approval of finished documents

# Acceptance Criteria

1. Generated drafts follow the structure of the selected template rather than returning only free-form prose.
2. Missing, empty, or invalid required sections are identified explicitly in the result.
3. Users can review structured output in the portal in a way that preserves section boundaries.
4. Validation results are returned in a machine-readable form that downstream UI or reporting can consume.
5. Template-specific logic does not bypass shared retrieval, routing, or trace conventions.

# Validation Notes

- tests should cover schema validation logic
- harness evals should include template-conformance scenarios

- Depends on `REQ-007` because structured drafting builds on the grounded Q and A flow and retrieval path already in place.
- harness evals should include template-conformance scenarios
