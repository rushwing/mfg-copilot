# User Story

As a platform engineer, I want a reproducible local development workflow so the team can build on a stable foundation.

# Goal

Establish the baseline local developer experience for the monorepo.

# Deliverables

- documented local startup commands for portal and core backend services
- repo-level scripts or make targets for bootstrap, dev, lint, and test entry points
- CI baseline that runs on pull requests
- container-friendly local runtime definition for shared dependencies

# In Scope

- service bootstrap commands
- local environment configuration templates
- baseline CI for lint, docs, and tests
- container-friendly local startup path

# Out of Scope

- full production deployment automation
- advanced release management

# Acceptance Criteria

1. A new developer can follow repo documentation to start the portal and core backend skeletons locally without undocumented manual steps.
2. Shared bootstrap and development commands exist for install, dev, lint, and test workflows.
3. Pull requests run a baseline CI workflow covering at least lint, docs validation, and repository smoke checks.
4. Shared dependency startup for local development is documented and container-friendly.
5. Required local configuration is represented by versioned templates rather than ad hoc machine-local setup.

# Validation Notes

- `tests/` should cover smoke-level startup and config validation
- CI should fail when required docs or config stubs are missing

# Dependency Notes

- No predecessor story is required because this is the foundation story for the repo itself.
