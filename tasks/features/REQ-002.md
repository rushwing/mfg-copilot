# User Story

As a manufacturing user, I want the system to know my role and context so responses and actions are scoped correctly.

# Goal

Create the identity and session baseline used by all later workflows.

# Deliverables

- auth entry points in portal and API gateway
- user session context model shared with downstream services
- role and manufacturing context resolver
- baseline RBAC enforcement hooks for read, approve, and execute classes

# In Scope

- auth entry point in portal and gateway
- session context propagation
- role, site, station, project, and phase context model

# Out of Scope

- complex delegated admin tooling
- execution approvals

# Acceptance Criteria

1. Authenticated requests carry resolved user identity through gateway-originated workflows.
2. Session context includes at least role, site, station, project, and phase fields where available.
3. Routing, audit, and approval-facing logic can consume the resolved context without re-deriving it ad hoc.
4. Unauthorized actions can be rejected consistently by policy at the API boundary.
5. The context contract is documented or typed so later services do not invent incompatible variants.

# Validation Notes

- `tests/` should cover auth/session parsing, RBAC decisions, and context propagation

# Dependency Notes

- Depends on `REQ-001` because auth and local session flows need the baseline app and service startup path in place.
- unit and integration tests should cover context parsing and RBAC decisions
