# User Story

As an agent developer, I want one retrieval surface so higher-level workflows do not depend on storage-specific logic.

# Goal

Prepare the retrieval platform used by doc, deployment, and ops workflows.

# Deliverables

- KB ingestion baseline for lifecycle documents and metadata
- Milvus indexing baseline
- Elasticsearch indexing baseline
- normalized retrieval interface in `rag-service`
- citation and provenance shape for higher layers

# In Scope

- KB ingestion pipeline baseline
- Milvus indexing baseline
- Elasticsearch indexing baseline
- normalized retrieval API with citations and provenance

# Out of Scope

- advanced ops triage ranking logic

# Acceptance Criteria

1. KB artifacts can be ingested with metadata required for filtering by project, product, phase, checkpoint, site, or station when applicable.
2. `rag-service` exposes one normalized retrieval contract for upper layers.
3. Retrieval responses include provenance or citation-ready fields.
4. Agent or orchestrator code paths are not required to call Milvus or Elasticsearch directly.
5. Retrieval foundation supports future hybrid or reranked strategies without changing upper-layer contracts.

# Validation Notes

- integration tests should cover ingestion and retrieval against real backing services where practical
- harness datasets should include retrieval relevance and citation checks

# Dependency Notes

- Depends on `REQ-003` because retrieval interfaces should be implemented on top of shared contracts, not invented independently.
- integration tests should cover ingestion and retrieval against real backing services where practical
- harness datasets should include retrieval relevance checks
