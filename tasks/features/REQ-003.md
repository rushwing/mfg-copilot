# User Story

As a platform engineer, I want stable shared contracts so services, portal views, and harnesses can evolve without drift.

# Goal

Stabilize the cross-service schemas needed before implementation fans out.

# Deliverables

- task envelope schema
- approval envelope schema
- retrieval request and response schema
- trace correlation contract with required IDs
- shared schema usage guidance for services and portal layers

# In Scope

- task envelope schema
- approval envelope schema
- retrieval request and response schema
- trace correlation metadata

# Out of Scope

- complete business logic for every workflow

# Acceptance Criteria

1. Shared schema files exist for the main platform envelopes used across gateway, orchestrator, agents, harness, and portal layers.
2. Approval, retrieval, and trace schemas define required identifiers and payload structure clearly enough for downstream implementation.
3. New service work references common contracts instead of inventing local envelope variants.
4. Trace identifiers align across gateway, orchestrator, harness, and observability documentation.
5. Schema examples or README guidance exist where future implementers would otherwise drift.

# Validation Notes

- schema fixture validation should run in CI where practical
- later service implementation should reference these schemas directly

# Dependency Notes

- Depends on `REQ-001` because shared contracts are only useful once the repo has a stable place and workflow to host them.
- schema linting or fixture validation should run in CI
