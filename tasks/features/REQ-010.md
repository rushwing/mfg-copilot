# User Story

As an approver or manager, I want generated summaries so I can understand document readiness and progress without manually gathering context.

# Goal

Provide reporting value alongside interactive doc workflows.

# Deliverables

- summary generation workflow for project, product, phase, checkpoint, or station scopes
- asynchronous or queued reporting path if needed
- portal retrieval and rendering for generated summaries
- artifact persistence for generated reports

# In Scope

- project and product summary jobs
- phase and checkpoint summary views
- daily, weekly, or monthly report generation patterns

# Out of Scope

- full BI dashboarding

# Acceptance Criteria

1. Summary generation can be triggered for supported scopes such as project, product, phase, checkpoint, or station.
2. Summaries reference grounded KB or enterprise context rather than unsupported generalization.
3. Generated reports are persisted as artifacts that can be viewed from the portal after completion.
4. Reporting workflows can run asynchronously where latency would otherwise block the user experience.
5. Generated summaries are auditable through task and trace metadata.

# Validation Notes

- reporting workflows should have asynchronous job and artifact tests

# Dependency Notes

- Depends on `REQ-009` so summary generation can build on readiness and structured context outputs rather than only on direct Q and A responses.
